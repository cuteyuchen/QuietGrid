# QuietGrid v2.0 设计文档包

> 面向币安 TradFi 永续合约休市窗口的、以市场状态识别和风险管理为核心的自适应网格交易系统设计方案。

## 1. 文档定位

本目录不是对现有代码的简单说明，而是以现有 QuietGrid 实现为基线，给出一套可分阶段落地的 v2.0 设计：

- 保留已经完成的时间窗口调度、交易所接入、选币、观察期、网格执行、状态机、风控、SQLite、FastAPI 与 Vue 3 控制台；
- 将原先以“固定规则网格”为中心的结构，升级为“市场状态识别 → 风险预算 → 自适应网格 → 库存管理 → 执行与验证”的闭环；
- 增加 Inventory Manager、Regime Score、亏损冷却、连续止损熔断、周末损失上限、保守成交仿真、Walk-Forward、策略重放和完整前端设计；
- 明确哪些是现有实现、哪些是建议修改、哪些必须经过回测与模拟盘验证后才能进入实盘。

## 2. 核心设计原则

1. **Risk First**：风险管理拥有最高优先级，任何策略指令都必须先经过风控。
2. **Regime Before Strategy**：先判断当前市场是否适合网格，再决定是否启动网格。
3. **Inventory Is Risk**：网格的主要风险不是单笔亏损，而是单边行情中不断累积的净仓位。
4. **Adaptive Instead of Static**：区间、间距、数量和每格仓位由市场状态与风险预算共同计算。
5. **Event Driven**：回测与实盘统一采用事件驱动流程，逐事件、逐 K 线推进，避免未来函数。
6. **Conservative Simulation**：回测宁可低估收益，也不要高估 Maker 成交概率。
7. **Everything Is Measurable**：每次启动、成交、击穿、止损、冷却与人工干预都必须可记录、可重放、可统计。
8. **No Automatic Risk Escalation**：盈利不能自动提高杠杆或扩大风险预算；风险升级必须经过独立验收。

## 3. 当前基线与 v2.0 目标

当前项目已经完成 M1–M9、Vue 3 + FastAPI 控制台、通知、回测、滚动重算、多账户和测试网烟测。v2.0 不要求推翻这些代码，而是通过新增模块和收紧接口逐步演进。

```text
当前：Scheduler → Observer → Grid Calculator → Grid Engine → Risk

目标：
Market Data → Feature Engine → Regime Engine
                            ↓
                      Risk Budget
                            ↓
              Adaptive Grid Generator
                            ↓
                 Inventory Manager
                            ↓
                    Risk Manager
                            ↓
                  Execution Engine
                            ↓
             Persistence / Replay / UI
```

## 5. 重要风险声明

- 本设计不能保证盈利，也不能消除极端行情、低流动性、交易所故障、政策变化和止损滑点风险。
- Maker 零费率、合约规则和交易时间必须由系统运行时查询验证，不能作为永久不变的事实写死。
- 原始方案中的 10 倍杠杆只应视为历史参数，不能作为 v2.0 实盘默认值。初期真实资金验收建议从 1 倍、单标的和极小风险预算开始；任何提高杠杆的决定都应基于长期样本外和小资金实盘数据。
- 本文档是一份软件与研究设计，不构成投资建议。
