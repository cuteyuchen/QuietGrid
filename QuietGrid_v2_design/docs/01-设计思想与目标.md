# 01. 设计思想与目标

## 1. 项目背景

QuietGrid 的原始假设是：美股真实市场休市期间，相关 TradFi 永续合约缺少主市场价格发现，价格更可能处于窄幅、低波动和均值回归状态；如果 Maker 成本足够低，网格可以通过重复成交积累有限利润。

这个假设值得研究，但不能直接当作永远成立的事实。系统必须能够回答三个问题：

1. 当前窗口是否仍然呈现可交易的均值回归状态？
2. 当前订单簿与交易成本是否允许网格产生正期望？
3. 一旦假设失效，系统能否在账户受到重大损伤前退出？

因此 v2.0 的核心不再是“把网格做得更密”，而是“识别优势存在、控制库存风险、在优势消失时停止”。

## 2. 项目目标

### 2.1 功能目标

- 自动识别 NYSE 休市窗口、半日市、节假日和夏令时变化；
- 自动筛选流动性、点差和历史稳定性合格的标的；
- 通过多指标 Regime Score 判断是否适合启用网格；
- 根据 ATR、实现波动率、成本下限、区间宽度和风险预算动态生成网格；
- 实时管理净库存、单向仓位、保证金占用和未配对成交；
- 提供分层风控：预防、降级、停止、强平兜底和冷却；
- 提供事件驱动回测、保守 Maker 成交模型、样本外验证和策略重放；
- 提供可观察、可审计、不能绕过交易进程安全检查的 Web 控制台。

### 2.2 质量目标

- **一致性**：回测、模拟和实盘尽可能复用相同的策略与风控代码。
- **可恢复性**：进程重启后能从交易所事实状态恢复，而不是只相信本地数据库。
- **可解释性**：每个启停决策都输出因素、阈值、分数和版本。
- **保守性**：不确定时不交易；成交无法确定时按最不利情况处理。
- **可测试性**：时间、行情、交易所和持久化均通过接口注入，支持确定性测试。

## 3. 非目标

v2.0 暂不追求：

- 高频做市或微秒级延迟；
- 预测单边涨跌；
- 依靠 LLM 直接自动下单；
- 自动寻找“最优参数”并立刻用于实盘；
- 通过高杠杆把小额账户快速翻倍；
- 在多个交易所之间进行跨所套利。

## 4. 核心设计原则

### 4.1 市场状态先于策略

Grid Engine 不拥有启动权。只有 Regime Engine 输出 `GRID_ALLOWED`，且 Risk Manager 批准风险预算后，Grid Generator 才能生成订单计划。

### 4.2 风险预算先于仓位

先确定本窗口最多允许损失多少，再反推每格数量和总名义仓位。禁止先决定杠杆与名义仓位，再寻找一个看似合理的止损。

### 4.3 库存风险独立建模

网格在震荡时看似市场中性，但成交后会产生净库存。库存暴露必须作为一级风险指标，而不是仅通过“价格是否击穿区间”间接处理。

### 4.4 不依赖意志力

人工操作必须经过受控命令队列、二次确认、风险复核和审计。参数提高风险等级时不能热加载到当前会话，必须从下一窗口生效，必要时还要经过审批标志。

### 4.5 数据与模型版本化

每次会话必须保存：数据版本、特征版本、规则版本、参数版本、代码提交号和交易所规则快照。否则无法可靠复现回测和实盘差异。

## 5. 关键假设及验证方式

| 假设 | 证据要求 | 失效处理 |
|---|---|---|
| 休市窗口波动较低且均值回归 | 历史窗口统计、半衰期、Hurst/ADF 辅助、突破率 | 禁止网格或缩小风险预算 |
| Maker 成本低 | 实际费率查询、资金费、点差、撤单率、成交后逆向选择 | 成本地板上调；不满足则跳过 |
| 挂单能以合理概率成交 | L1/L2 数据或保守成交模型；实盘 fill ratio | 降低预期收益或扩大间距 |
| 止损能成交 | 深度压力测试、最差滑点估计、交易所端止损验收 | 降低仓位；深度不足不交易 |
| 多标的能够分散风险 | 相关性和共同因子暴露分析 | 对相关标的共享总风险上限 |

## 6. 成功标准

策略是否成功不能只看累计收益。至少同时满足：

- 样本外净收益为正；
- 最大回撤在预设风险预算内；
- 单次最坏窗口不会摧毁多个窗口利润；
- Maker 成交假设与实盘差异可解释；
- 无人工干预和有人工干预的收益差异可统计；
- 进程故障、断网和 API 异常下不会遗留不可控仓位；
- 真实盘扩大资金前，至少完成多个独立窗口的小资金验收。
