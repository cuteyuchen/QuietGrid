# 03. 市场状态识别（Regime Detection）

## 1. 目标

Regime Engine 的任务不是预测下一分钟涨跌，而是判断当前环境是否适合运行均值回归网格，并给出可解释的置信度。

建议状态：

- `QUIET_RANGE`：低至中等波动、弱趋势、流动性合格，允许网格；
- `TREND_UP` / `TREND_DOWN`：存在持续方向，禁止常规双向网格；
- `VOLATILE`：波动快速扩大，停止或禁止网格；
- `ILLIQUID`：点差、深度或撤单率异常；
- `EVENT_RISK`：存在已知复市、重大公告或外部风险；
- `UNKNOWN`：数据不足或指标冲突，不交易。

## 2. 特征集合

### 2.1 波动率

- 1 分钟对数收益滚动标准差；
- ATR / Price；
- Parkinson 或 Garman–Klass 波动率；
- 短窗与长窗波动率比值，识别突然扩张。

```text
vol_expansion = vol_short / max(vol_long, epsilon)
```

### 2.2 趋势

- EMA 斜率标准化；
- ADX；
- 最近 N 分钟线性回归斜率 / 残差波动；
- 单向 K 线比例；
- 中枢漂移速度。

低波动并不等于无趋势，因此趋势门控必须独立存在。

### 2.3 流动性与交易成本

- bid-ask spread；
- 前 5/10 档深度；
- 盘口不平衡；
- 订单撤单率；
- Maker 实际成交率；
- 资金费率与下一次资金费结算时间；
- 实际 Maker/Taker 费率查询结果。

### 2.4 均值回归特征

- 价格偏离 EWMA 中枢后的回归速度；
- Ornstein–Uhlenbeck 简化估计的半衰期；
- Hurst 指数仅作为辅助，不作为单独开关；
- 观察期突破次数和假突破比例。

### 2.5 时间与事件风险

- 距下次盘前/常规开盘时间；
- 半日市与节假日；
- 资金费结算窗口；
- 交易所维护公告；
- 可选的 AI/新闻风险评分，但只能作为门控特征，不能直接下单。

## 3. 分数设计

建议先使用透明的规则评分，而不是直接训练复杂模型。

```text
grid_score =
    0.25 * volatility_score
  + 0.25 * trend_score
  + 0.20 * liquidity_score
  + 0.15 * mean_reversion_score
  + 0.10 * cost_score
  + 0.05 * event_score
```

每个子分数为 0–100，100 表示更适合网格。

建议门控：

```text
hard_block =
    time_window_invalid
    OR spread > spread_hard_limit
    OR vol_expansion > vol_hard_limit
    OR depth < depth_min
    OR data_stale
    OR maker_fee_unverified
    OR event_risk_hard_block

allowed = NOT hard_block AND grid_score >= enter_threshold
```

使用双阈值避免频繁启停：

- 进入阈值 `enter_threshold = 75`；
- 保持阈值 `stay_threshold = 65`；
- 当运行中分数落到 65 以下，进入风险降级或退出；
- 恢复时必须重新达到 75，并持续若干分钟。

## 4. 标准化方法

不要直接对不同标的使用相同绝对波动阈值。建议使用历史休市窗口的滚动分位数：

```text
volatility_score = 100 - percentile_rank(current_vol, historical_weekend_vol)
```

流动性指标可用横截面和自身历史双重标准化，避免“全市场都很差时仍选出相对第一名”。

## 5. 输出解释

每次决策必须输出：

```json
{
  "state": "QUIET_RANGE",
  "grid_score": 82.4,
  "allowed": true,
  "reasons": [
    "短窗波动率位于历史休市窗口第31百分位",
    "EMA斜率接近0，ADX=13.2",
    "点差0.018%，前5档深度满足最低要求",
    "距离盘前强制离场仍有11.6小时"
  ],
  "hard_blocks": [],
  "model_version": "regime-rules-v2.0.0"
}
```

## 6. 恢复确认

冷却后不能因为某一根 1 分钟 K 线恢复就立即重启。建议同时满足：

- 最短冷却时间结束；
- `grid_score >= enter_threshold` 连续 15 分钟；
- 短窗波动率低于长窗波动率；
- 价格重新围绕新中枢横盘；
- 库存与挂单均为 0；
- 本窗口累计止损次数未达到上限。

## 7. AI Agent 的位置

Trading Agent 可以作为 `event_score` 或解释模块输入，例如输出宏观、新闻和流动性风险观点。但必须满足：

- 不直接调用 Execution；
- 置信度需要历史校准；
- 任何 LLM 输出失败、超时或格式异常时，系统回退到规则模型；
- LLM 不可使用在当时尚未发布的信息进行回测；
- AI 只可以让系统更保守，初期不允许单独把禁止状态改为允许。
