# 05. 库存风险管理（Inventory Manager）

## 1. 为什么必须独立建模

网格会在价格下跌时连续买入、上涨时连续卖出。即使初始挂单对称，成交后也会形成净多或净空库存。趋势出现时，库存风险往往比单格收益大得多。

## 2. 核心指标

### 2.1 净名义仓位

```text
net_notional = net_qty * mark_price
```

### 2.2 库存利用率

```text
inventory_utilization = abs(net_notional) / max_inventory_notional
```

### 2.3 方向偏置

```text
inventory_bias = net_notional / gross_notional_limit
```

范围约为 -1 到 1。

### 2.4 未配对库存

需要区分已完成的一买一卖网格对与尚未匹配的成交。未配对库存必须记录来源格位和预期退出格位。

### 2.5 库存风险分数

```text
inventory_risk =
    0.45 * utilization_score
  + 0.25 * unrealized_loss_score
  + 0.20 * trend_alignment_score
  + 0.10 * time_to_close_score
```

当净多库存遇到下跌趋势时，`trend_alignment_score` 应迅速恶化。

## 3. 风险等级与动作

| 等级 | 库存利用率示例 | 动作 |
|---|---:|---|
| NORMAL | < 40% | 正常挂单 |
| CAUTION | 40%–60% | 减少同向新单数量 |
| HIGH | 60%–80% | 禁止继续增加同向库存，只允许减仓单 |
| CRITICAL | > 80% 或止损风险超预算 | 撤单、主动降仓或关闭会话 |

阈值必须通过回测验证，表中只表示机制。

## 4. 同向挂单抑制

当净多库存偏高：

- 取消最远端新增买单；
- 降低剩余买单数量；
- 保留或适度增加卖出减仓单，但不得通过跨价挂单破坏 POST_ONLY 约束；
- 若趋势与库存同向恶化，直接进入降仓或冷却。

## 5. 主动再平衡

主动再平衡不是无条件市价平仓。动作优先级：

1. 取消增加风险的挂单；
2. 用被动订单减仓；
3. 若风险继续恶化或接近强制离场，使用限价 IOC/市价 reduce-only；
4. 达到硬止损后无条件退出。

## 6. 多标的共享风险

多个美股科技代币可能高度相关。不能把每个标的的 20% 风险简单相加当作分散。

建议建立相关性组：

```yaml
risk_groups:
  mega_cap_tech:
    symbols: [AAPLUSDT, MSFTUSDT, NVDAUSDT, METAUSDT]
    max_group_notional_pct: 0.20
```

组内共享最大库存与最大损失预算。

## 7. 数据结构

```python
@dataclass
class InventoryLot:
    lot_id: str
    session_id: int
    side: str
    entry_price: Decimal
    qty: Decimal
    entry_grid_index: int
    target_exit_price: Decimal
    opened_at: datetime
    status: str

@dataclass
class InventorySnapshot:
    net_qty: Decimal
    net_notional: Decimal
    gross_notional: Decimal
    avg_entry_price: Decimal | None
    unrealized_pnl: Decimal
    utilization: float
    risk_score: float
    unpaired_lots: int
```

## 8. 恢复与对账

进程启动或 WebSocket 断线后：

1. 拉取交易所持仓与所有挂单；
2. 与本地 lots 和 orders 对比；
3. 无法匹配的真实仓位标记为 `ORPHAN_POSITION`；
4. 在恢复完成前禁止新增风险；
5. 若无法安全重建配对关系，执行受控平仓，而不是假设库存为零。
