# 06. 风险管理与冷却机制

## 1. 风控层级

```text
L0 交易前门控：时间、数据、费率、流动性、Regime
L1 风险预算：账户、窗口、标的、相关性组、库存
L2 运行降级：缩量、禁挂同向单、扩大间距
L3 会话退出：击穿、止损、止盈、人工关闭
L4 全局熔断：连续亏损、每日/周末损失、系统故障
L5 交易所兜底：STOP_MARKET / reduce-only / 强制清扫
```

## 2. 风险预算

建议用账户权益比例定义，而不是固定“每标的 200 USDT + 10 倍”。

```text
window_loss_budget = equity * max_weekend_loss_pct
session_loss_budget = min(
    equity * max_session_loss_pct,
    remaining_window_budget,
    group_remaining_budget
)
```

所有未实现最坏损失与已实现亏损之和不得超过预算。

## 3. 止损类型

### 3.1 区间击穿

价格突破网格区间并不等同于硬止损。区间击穿后应：

1. 冻结新增风险；
2. 撤销挂单；
3. 尽快平掉库存；
4. 进入冷却；
5. 重新观察并计算新中枢。

### 3.2 会话损失止损

```text
session_pnl <= -session_loss_budget → CLOSING
```

### 3.3 硬价格止损

交易所端 STOP_MARKET 仅作为程序外兜底。代码端应更早触发受控退出，交易所端价格可稍宽，但不能宽到接近爆仓线。

### 3.4 时间止损

距离强制离场过近时，即使盈利目标未完成也必须关闭。新增网格应设最小剩余时间要求。

## 4. 冷却机制

### 4.1 触发条件

- 区间击穿；
- 会话止损；
- 短窗波动率异常；
- 订单簿流动性快速恶化；
- 多次 POST_ONLY 拒单或成交异常；
- 人工暂停。

### 4.2 冷却状态

进入冷却前必须确认：

- 普通挂单已撤销；
- 条件单状态已确认；
- 净持仓为零或进入专门的 `CLOSING`；
- 触发原因、损失和市场快照已保存。

### 4.3 最短冷却

不同触发原因使用不同冷却：

| 原因 | 建议机制 |
|---|---|
| 轻微波动超阈值 | 15–30 分钟 + 连续恢复确认 |
| 区间击穿 | 30–60 分钟 + 新观察期 |
| 会话止损 | 本标的至少 2 小时或本窗口禁用 |
| 连续止损 | 本窗口全局停止 |
| 系统/数据异常 | 人工确认后恢复 |

### 4.4 恢复条件

```text
minimum_cooldown_elapsed
AND flat_position
AND no_open_orders
AND data_fresh
AND regime_score >= enter_threshold for confirm_minutes
AND volatility_recovered
AND remaining_time >= min_runtime
AND loss_limits_not_breached
```

## 5. 连续亏损熔断

建议同时统计：

- 连续亏损会话数；
- 同一标的连续止损次数；
- 本窗口触发冷却次数；
- 人工违背风控次数。

示例规则：

```text
symbol_stop_count >= 2 → 本窗口禁用该标的
window_stop_count >= 3 → 停止全部新开仓
consecutive_losing_windows >= 3 → 自动进入研究模式，需人工复核
```

## 6. 每日与周末损失上限

```text
realized_pnl + conservative_unrealized_pnl <= -window_loss_budget
```

触发后：

- 取消所有挂单；
- 平掉全部仓位；
- 停止本窗口；
- 锁定高风险配置修改；
- 发送高优先级通知；
- 生成自动事故报告。

## 7. 杠杆治理

- 杠杆是风险放大器，不是策略收益来源；
- 杠杆不得由前端随时热修改到当前会话；
- 盈利后禁止自动提高杠杆；
- 实盘初期默认 1 倍；
- 只有在足够的样本外、模拟和小资金实盘通过后，才允许在下一窗口提高上限；
- 即使交易所设置了较高杠杆，系统也应通过名义仓位和损失预算保持更低的有效杠杆。

## 8. 人工控制规则

人工操作也必须经过风控：

- “开始网格”只是请求，不保证执行；
- “恢复”前重新运行 Regime 和 Risk Check；
- “平仓”可以无条件执行，但必须二次确认；
- 提高风险参数需要延迟生效和审计；
- 所有控制命令携带用户、时间、来源、理由和幂等键。

## 9. 事故报告

每次硬止损、全局熔断或异常仓位都自动生成：

- 时间线；
- 当时市场特征；
- 风控阈值；
- 订单与成交；
- 理论损失与实际损失差异；
- 滑点与流动性；
- 代码版本和配置版本；
- 是否存在人工干预。
