# 07. 回测与验证

## 1. 目标

回测不是证明策略会赚钱，而是尽可能排除明显无效、未来函数、成本遗漏和尾部风险。

## 2. 事件驱动原则

回测按真实时间推进：

```text
收到已完成的 1m Bar
→ 更新特征
→ 计算 Regime
→ 生成订单意图
→ Risk Check
→ 订单进入模拟撮合
→ 下一事件
```

在当前 Bar 完成后计算的信号，最早只能在下一可交易时点执行。

## 3. 未来函数防护

### 3.1 数据时间字段

每条数据至少有：

- `event_time`：市场事件发生时间；
- `available_time`：系统可获得该数据的时间；
- `ingested_time`：系统接收时间。

策略只能使用 `available_time <= decision_time` 的数据。

### 3.2 常见错误

- 用当前 K 线的最终高低价决定该 K 线内订单；
- 用日终修正后的资金流进行盘中决策；
- 用发布日期而非实际发布时间处理公告；
- 选币时使用未来 24 小时成交额；
- 参数优化后在同一历史区间报告绩效；
- 用最终下架名单清洗早期数据导致幸存者偏差。

### 3.3 自动检测

回测引擎应在特征读取时验证 `as_of_time`，发现引用未来数据立即抛出 `LookAheadViolation`，整个回测标记无效。

## 4. Maker 成交模型

仅用 1 分钟 K 线高低价判断“触价即成交”会明显高估收益。分层实现：

### Level 0：最保守 K 线模型

- 价格必须穿过挂单一个最小 tick 才视为可能成交；
- 同一根 K 线同时触及多个层级时，按最不利路径处理；
- 同根 K 线触及止损和盈利单，止损优先；
- 限制每根 K 线最大成交层数；
- 对成交概率应用折扣。

### Level 1：L1 模型

使用 bid/ask、spread 和成交方向，挂单只有在对应一侧被主动成交量覆盖时才成交。

### Level 2：L2 队列模型

根据挂单前方队列量、市场成交量和撤单估计排队位置。只有足够成交量穿透队列才视为成交。

实盘前至少应使用 Level 1；如果拿不到历史盘口，则对 Level 0 做大幅保守折扣。

## 5. 成本模型

必须包括：

- 实际 Maker/Taker 费率；
- 资金费；
- 市价止损滑点；
- 价格精度和数量取整；
- POST_ONLY 拒单造成的机会损失；
- 断线期间无法补单；
- 重新建网格的撤单与成交延迟；
- 低流动性压力滑点。

## 6. 数据切分

### 6.1 时间序列切分

禁止随机打乱。建议：

```text
开发集 → 验证集 → 完全未触碰的测试集
```

### 6.2 Walk-Forward

示例：

```text
窗口1：过去 12 个月校准，未来 3 个月测试
窗口2：向前滚动 3 个月
...
```

每个测试窗口只能使用此前数据选择参数。

## 7. 参数治理

- 参数搜索范围必须事先定义；
- 优先选择宽阔稳定的平台区，而不是单点最优；
- 记录所有实验，避免只保留成功结果；
- 参数越多，需要越强的复杂度惩罚；
- 真实盘参数只能来自已冻结的实验版本。

## 8. Monte Carlo 与压力测试

对成交顺序、滑点、漏单和波动进行重采样：

- 打乱会话收益顺序；
- 提高止损滑点；
- 随机丢失一定比例 Maker 成交；
- 增加 API 延迟；
- 模拟周末突发单边 3%、5%、10%；
- 模拟 WebSocket 断线和进程重启；
- 模拟费率活动结束。

输出收益分布而不是单一曲线。

## 9. 关键指标

### 收益与回撤

- 净收益、年化仅作辅助；
- Max Drawdown；
- Calmar、Sortino、Profit Factor；
- 最坏单窗口、CVaR；
- 回撤恢复时间。

### 网格专用指标

- Grid Fill Ratio；
- Pair Completion Ratio；
- 单格净收益；
- 每窗口成交密度；
- 未配对库存最大值；
- Inventory Utilization P95/P99；
- Breakout Loss / Grid Profit 比值；
- 冷却次数、连续止损次数。

### 执行指标

- POST_ONLY 拒单率；
- 模拟与实盘成交差；
- 平仓滑点；
- 对账差异次数；
- 数据延迟和断线时间。

## 10. 策略晋级制度

```text
L0 单元测试
L1 历史回测
L2 Walk-Forward 样本外
L3 纸面交易 / Mock 实时
L4 测试网链路验收
L5 极小资金、1倍、单标的
L6 多窗口稳定后小幅扩展
```

任何一级失败，退回研究阶段。禁止因为短期盈利跳级。
