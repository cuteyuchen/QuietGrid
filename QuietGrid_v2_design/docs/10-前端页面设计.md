# 10. 前端页面设计

## 1. 产品目标

前端不是“收益展示页”，而是交易系统控制中心。优先级：

1. 当前是否安全；
2. 系统为什么运行或停止；
3. 库存和风险是否接近上限；
4. 真实订单与本地状态是否一致；
5. 操作能否经过审计和二次确认。

技术栈延续 Vue 3 + Vite + TypeScript。建议使用 Pinia 管理前端状态，ECharts 或 Lightweight Charts 展示图表。

## 2. 信息架构与路由

```text
/                       总览 Dashboard
/market-regime          市场状态
/sessions               会话列表
/sessions/:id           会话详情与网格
/sessions/:id/inventory 库存详情
/risk                   风险中心
/orders                  订单与成交
/backtests               回测中心
/backtests/:id           回测报告
/replay/:sessionId       策略重放
/logs                    日志与告警
/settings                参数与账户设置
/operations              运维与安全清扫
```

## 3. 全局布局

```text
┌──────────────────────────────────────────────────────────────┐
│ QuietGrid | TESTNET | Trader: RUNNING | Data: HEALTHY | 用户 │

│ Dashboard    │                                               │
│ Market       │                主内容区域                      │
│ Sessions     │                                               │
│ Risk         │                                               │
│ Orders       │                                               │
│ Backtests    │                                               │

```

顶部必须永久显示环境。真实盘使用醒目边框与文字，不仅依赖颜色。

## 4. Dashboard 页面

### 4.1 第一行：全局安全卡片

- 当前环境；
- 距强制离场时间；
- 交易进程状态；
- 数据新鲜度；
- 交易所连接；
- 全局风险等级。

### 4.2 第二行：账户与窗口

```text
账户权益 | 可用余额 | 当前名义仓位 | 有效杠杆
本窗口已实现PnL | 保守未实现PnL | 剩余损失预算 | 活跃会话
```

### 4.3 第三行：活跃会话卡片

每张卡片显示：

- 标的、状态和 Regime Score；
- 当前价格、区间和网格数量；
- Inventory Utilization；
- 当前风险等级；
- 已实现/未实现盈亏；
- 距自动重算和强制离场时间；
- “查看详情”和“请求关闭”。

### 4.4 告警区域

只显示需要行动的告警：交易所对账差异、条件止损缺失、数据过期、风险预算不足、Web 未启用鉴权等。

## 5. Market Regime 页面

### 5.1 主要组件

- `Grid Score` 仪表；
- 当前状态徽标；
- 波动率、趋势、流动性、成本、均值回归和事件风险六个子分数；
- 最近 24 小时分数曲线；
- 允许/禁止原因列表；
- 数据时间与模型版本。

### 5.2 解释面板

```text
允许网格：是
主要正面因素：趋势弱、点差低、波动率处于历史中低分位
主要风险：前5档深度较上一窗口下降18%
恢复/退出阈值：Enter 75 / Stay 65
```

禁止仅显示“置信度 82%”而没有校准含义。

## 6. 会话与网格详情页

### 6.1 网格可视化

价格图叠加：

- 当前中枢；
- 上下沿；
- 每个网格价位；
- 当前挂单；
- 已成交库存 lot；
- 硬止损；
- 当前价格；
- 网格版本变更时间。

点击某个格位显示订单、成交、数量、配对状态和预计净收益。

### 6.2 会话状态时间线

```text
OBSERVING → READY → RUNNING → REBALANCING → RUNNING → COOLDOWN
```

每次转换展示触发原因和快照链接。

### 6.3 操作

- 暂停新增风险；
- 请求恢复；
- 请求关闭会话；
- 下载事故/会话报告。

“关闭”使用输入标的代码二次确认。恢复按钮在风控不允许时禁用，并展示原因。

## 7. Inventory 页面

### 7.1 概览

- 净数量与净名义仓位；
- 平均成本；
- 未实现盈亏；
- Inventory Utilization；
- 风险分数；
- 未配对 lot 数量；
- 距硬止损的估计损失。

### 7.2 Lot 表格

| 开仓时间 | 方向 | 价格 | 数量 | 来源格位 | 目标退出 | 状态 | 浮盈亏 |

支持按风险和开仓时间排序。

### 7.3 风险图

展示库存利用率随时间变化，并叠加 CAUTION/HIGH/CRITICAL 阈值。

## 8. Risk Center 页面

### 8.1 风险预算树

```text
账户预算
└── 周末窗口预算
    ├── 相关性组预算
    │   └── 会话预算
    └── 其他标的
```

展示每层上限、已用、剩余和最坏损失估计。

### 8.2 熔断器

- 当前状态；
- 触发原因；
- 冷却剩余时间；
- 连续亏损次数；
- 本窗口止损次数；
- 恢复条件进度。

### 8.3 风险操作

- 停止所有新开仓；
- 平掉所有仓位；
- 安全清扫；
- 锁定配置。

这些按钮必须固定在页面显眼位置，但使用危险操作样式和二次确认。

## 9. Orders 页面

提供三个视图：交易所真实挂单、本地订单投影、差异列表。任何差异都可以展开查看对账过程。部分成交必须单独显示，不能只分 OPEN/FILLED。

## 10. Backtest 页面

### 10.1 回测列表

显示数据区间、代码版本、配置版本、成交模型、净收益、最大回撤、Profit Factor、P99 库存和状态。

### 10.2 回测详情

- 权益与回撤曲线；
- 窗口收益分布；
- 网格利润与突破损失分解；
- Inventory P50/P95/P99；
- Regime 混淆与过滤效果；
- 成交率与成本敏感性；
- Walk-Forward 各折表现；
- Monte Carlo 分布。

必须醒目标注“样本内/样本外”和成交模型等级。

## 11. Strategy Replay 页面

策略重放是 v2.0 的关键调试功能。

布局：

```text
左：K线、网格、订单与库存动画
右上：Regime 与 Risk 快照
右中：事件时间线
右下：日志、订单与状态变化
底部：播放、暂停、倍速、逐事件、跳转
```

支持回放历史实盘和回测，并比较“实际动作”与“假设严格遵守系统动作”。

## 12. Logs 页面

- 按级别、模块、标的、会话和时间筛选；
- 将结构化字段展开；
- ERROR 关联事故报告；
- 支持导出；
- 避免只显示无结构纯文本。

## 13. Settings 页面

参数按层级分组：

- 账户与环境；
- 风险预算；
- Regime；
- Grid；
- Cooldown；
- 选币；
- 通知；
- Web 安全。

显示当前激活版本、草稿版本和差异。提高风险的字段标记“下窗口生效，需验证”。

## 14. Operations 页面

- 连接与权限检查；
- 持仓/订单烟测；
- Maker 费率健康；
- 条件止损链路；
- 安全清扫；
- 数据库备份与恢复测试；
- 进程启动/停止状态；
- 最近一次验收记录。

## 15. 响应式与可访问性

桌面端为主要目标，移动端只保证查看与紧急关闭，不在手机上提供高风险参数编辑。状态不能仅用颜色区分；所有图表提供数值和文本说明。

## 16. 前端状态管理

Pinia store 建议：

```text
systemStore
accountStore
sessionStore
regimeStore
riskStore
orderStore
backtestStore
notificationStore
```

SSE 仅触发资源刷新；危险操作采用独立 composable，统一处理二次确认、幂等键和命令状态轮询。
