# 11. 部署、运维与安全

## 1. 环境分离

至少区分：

- `development`：Mock；
- `testnet`：测试交易所链路；
- `paper`：真实行情、模拟成交；
- `live`：真实资金。

环境名必须同时显示在日志、数据库、通知和前端顶部。数据库与密钥不能跨环境复用。

## 2. 启动前检查

1. 配置与 schema 校验；
2. API 权限校验，明确禁止提现权限；
3. 时间同步；
4. 代理和 REST/WS 连通；
5. 账户持仓与挂单检查；
6. Maker/Taker 费率与交易规则；
7. 条件止损接口；
8. 数据库写入与备份；
9. Web 鉴权；
10. 强制离场时间计算。

任一关键项失败，不能启动新仓。

## 3. 故障恢复

### 3.1 进程崩溃

systemd 可自动重启，但重启后先进入 `RECOVERING`：拉取交易所挂单、持仓、条件单和最近成交；完成对账前不得开新仓。

### 3.2 WebSocket 断线

- 立即标记数据不健康；
- 暂停新增风险；
- 使用 REST 拉取真实状态；
- 重连成功后补齐事件；
- 断线超过上限则关闭会话。

### 3.3 数据库不可用

不能继续“只在内存中交易”。进入降级模式、撤单并视情况平仓。

### 3.4 交易所写请求超时

下单超时状态未知时不能直接重试新订单。先用 client order id 查询结果，再决定是否重试。

## 4. 安全

- 密钥仅存环境变量或 Secret Manager；
- API Key 不启用提现；
- Web 必须 token 或更强鉴权；
- 反向代理和 TLS；
- 防火墙只允许可信来源；
- 前端构建产物不包含秘密；
- 日志脱敏；
- 控制命令记录操作者；
- 高风险操作二次确认。

## 5. 告警分级

### INFO

窗口开始、观察完成、网格启动、正常成交。

### WARN

Regime 降级、库存 CAUTION、POST_ONLY 拒单率升高、费率变化、短暂断线。

### ERROR

止损、库存 CRITICAL、条件止损缺失、持续断线、对账差异、数据库错误。

### CRITICAL

残留未知仓位、无法平仓、全局损失熔断、实盘环境安全配置缺失。

CRITICAL 应通过多个通道通知。

## 6. 可观测性

核心指标：

- REST/WS 延迟和断线；
- 行情数据 age；
- 订单确认延迟；
- 对账差异；
- 条件止损覆盖率；
- 事件队列长度；
- 数据库写入延迟；
- 活跃会话和名义仓位；
- 风险预算利用率；
- 冷却与熔断状态。

## 7. 备份与恢复

- 每周自动备份数据库和配置版本；
- 备份加密；
- 定期在独立目录执行恢复演练；
- 事故报告和实盘成交记录额外归档；
- 所有报告记录代码提交号。
