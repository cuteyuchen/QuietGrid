# 14. 配置参数规范

## 1. 参数分层

### 1.1 运行配置

环境、账户、代理、数据库、日志和通知。修改不改变策略逻辑。

### 1.2 风险配置

账户/窗口/会话损失上限、最大库存、并发、有效杠杆和冷却。必须有严格验证和审计。

### 1.3 策略配置

Regime 权重、区间、间距、格数和观察窗口。必须版本化并经过回测。

### 1.4 实验配置

仅用于研究，禁止直接加载到 live。

## 2. 默认值原则

- 以不交易和低风险为默认；
- 缺失关键参数时启动失败；
- 费率、交易规则和时间动态查询；
- 10 倍杠杆不得作为 live 默认；
- 最大损失使用权益比例，同时允许绝对金额硬上限；
- 参数必须带单位，例如 `_pct`、`_minutes`、`_usdt`。

## 3. 热加载

可以在当前会话立即生效：

- 降低资金上限；
- 降低并发；
- 暂停新开仓；
- 提高保守程度。

只能下一窗口生效：

- 提高杠杆；
- 提高损失上限；
- 扩大库存；
- 降低 Regime 门槛；
- 缩小成本地板；
- 改变网格算法版本。

## 4. 配置版本状态

```text
DRAFT
→ BACKTESTED
→ VALIDATED
→ ACTIVE_TESTNET
→ ACTIVE_LIVE
→ RETIRED
```

每次激活记录审批理由、回测报告和代码版本。

## 5. 参数敏感性

报告至少展示：

- `enter_threshold`；
- `step_pct` 或其系数；
- `max_grid_num`；
- `max_inventory_pct`；
- `session_loss_pct`；
- `cooldown_minutes`；
- `stop_buffer_pct`。

参数稍有变化就从盈利变巨亏，说明策略不稳健，不能进入实盘。
