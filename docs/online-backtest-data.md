# 在线历史数据与回测中心

本文说明 QuietGrid 当前已经落地的在线历史数据、冻结数据集、NYSE 休市窗口回测和前端操作流程。详细的原始需求与算法讨论保留在项目外部设计资料中；本文只描述当前代码可验证的行为。

## 1. 设计目标

回测中心不是“选个 CSV 然后看收益”的简化页面，而是一条完整、可复现、可审计的研究链路：

```text
选择数据来源
  → 检查数据范围与预计体量
  → 下载 / 上传
  → 标准化与质量校验
  → 切分 NYSE 休市窗口
  → 冻结 Dataset ID 与 SHA-256
  → 配置保守回测参数
  → 执行回测
  → 查看概览、验证、执行与成本
```

核心原则：

- 易用性来自流程、默认值、分组和解释，不靠删除专业功能；
- 同一冻结数据集在同一代码和参数下应可复现；
- Risk Manager 和成交假设优先于收益展示；
- NYSE 休市窗口必须独立观察、独立建网格、独立结算；
- 不具备历史盘口或真实资金费序列时，不生成伪精度结论。

## 2. 前端工作流

回测中心按任务顺序组织为三个阶段。

### 2.1 数据准备

提供三种入口：

1. **Binance 在线**：搜索 USDⓈ-M 永续合约，选择周期、时间范围和窗口模式，先预估数据量，再创建后台下载任务；
2. **已有数据集**：搜索并复用冻结数据集或兼容旧版 CSV；
3. **上传 CSV**：上传后先执行标准化、质量校验和冻结，不直接进入回测。

下载任务显示阶段、进度、页数和已下载行数，可取消。完整阶段包括下载、标准化、质量校验和 NYSE 窗口切分。

选中数据集后显示 Dataset ID、实际范围、K 线数量、休市窗口数、Provider、Checksum 和完整质量报告。冻结数据集可在同一区域删除；删除需要二次确认，已经被回测报告引用的数据集由服务端拒绝删除。

### 2.2 参数与启动

常用参数直接可见，成交、费用、Walk-Forward、Monte Carlo 和样本治理参数保留在明确分组中。数据集选择会同步标的；服务端再次校验 Dataset ID 与回测标的一致，避免误用错误行情。

当前保留的核心能力包括：

- 观察行数、资本和杠杆；
- `L0_CONSERVATIVE` 保守成交；
- Maker 成交概率、每 Bar 最大成交层数；
- Maker/Taker 费用、止损滑点和可选资金费假设；
- Walk-Forward 测试长度；
- Monte Carlo 模拟次数、漏单和亏损放大；
- 固定窗口分布；
- `DEVELOPMENT`、`VALIDATION`、`OOS_FROZEN` 样本标签和参数冻结确认。

### 2.3 结果分析

报告没有压缩成单一收益数字，而是分为：

- **概览**：权益、回撤、库存利用率、核心指标和成本分解；
- **验证**：Walk-Forward、Monte Carlo、窗口收益分布、成本敏感性和 NYSE 逐窗口结果；
- **执行与成本**：成交模型、成交明细、运行参数、数据与代码元数据。

长表默认折叠，用户需要时再展开；核心指标和风险结论始终可见。页面同时适配桌面和移动端，移动端不横向溢出。

## 3. 数据集模型

冻结数据集至少保存：

- `dataset_id`、Provider、Symbol、Interval；
- 请求和实际时间范围；
- `window_mode`、窗口数量；
- 标准化后的 K 线文件；
- SHA-256 Checksum；
- Schema Version；
- 行数、字节数和质量报告；
- 创建状态、任务进度和错误信息。

质量校验覆盖字段、时间顺序、重复与冲突重复、K 线闭合、缺口、OHLC 合法性和未闭合 Bar。状态为 `READY` 或 `READY_WITH_WARNINGS` 才允许回测；文件校验和不匹配时会标记为损坏并拒绝使用。

## 4. NYSE 休市窗口语义

`NYSE_CLOSED_ONLY` 使用真实 NYSE 日历和 `America/New_York` 时区，自动处理周末、节假日、半日市和夏令时。

一个窗口定义为：

```text
本次 NYSE 收盘
  → 下一交易日 04:00（纽约时间）之前
  → 再提前 force_close_minutes（默认 120 分钟）强制离场
```

实现约束：

- 每个窗口独立取观察段并计算网格；
- 观察数据不足或可交易数据不足时，窗口被跳过并记录原因；
- 达到强制离场边界前撤单并按保守 Taker 成本平仓；
- 仓位、挂单和权益不会跨窗口继承；
- 主回测窗口分布来自独立窗口结果，不把多个周末拼成一段连续交易。

`RAW_RANGE` 保留完整原始范围，供兼容旧版研究或非休市窗口实验使用。

## 5. API

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/api/v2/backtest-data/providers` | 数据源能力 |
| `GET` | `/api/v2/backtest-data/providers/{provider}/symbols` | 标的搜索 |
| `POST` | `/api/v2/backtest-data/preview` | 数据量与缓存预估 |
| `POST` | `/api/v2/backtest-data/jobs` | 创建下载和冻结任务 |
| `GET` | `/api/v2/backtest-data/jobs/{job_id}` | 读取任务进度 |
| `POST` | `/api/v2/backtest-data/jobs/{job_id}/cancel` | 请求取消并写审计 |
| `POST` | `/api/v2/backtest-data/upload` | 上传、校验并冻结 CSV |
| `GET` | `/api/v2/backtests/datasets` | 数据集列表 |
| `GET` | `/api/v2/backtests/datasets/{dataset_id}` | 数据集与窗口详情 |
| `DELETE` | `/api/v2/backtests/datasets/{dataset_id}` | 软删除未被引用的数据集 |
| `POST` | `/api/v2/backtests` | 使用 Dataset ID 或旧版 CSV 启动回测 |
| `GET` | `/api/v2/backtests` | 回测列表 |
| `GET` | `/api/v2/backtests/{run_id}` | 报告详情 |

下载任务使用服务端线程池执行，前端目前用短轮询显示进度；这是接口契约允许的降级实现，不影响任务的持久化、取消和进程重启后的中断恢复。

## 6. 安全与审计

- 在线数据源固定为公共只读行情接口，不接收 API Key；
- Provider、Symbol、Interval、时间范围和上传文件大小均有白名单或边界校验；
- 上传临时文件与冻结文件限制在配置目录内；
- 上传、取消和删除均写入 `audit_logs`；
- 删除采用软删除，且有回测引用时返回冲突；
- Dataset ID 与回测 Symbol 不一致时返回 422；
- 同一账户并发数据任务受限，缓存命中会复用已冻结数据。

## 7. 验证基线

自动测试覆盖：

- Binance 单页、多页、短尾页、空数据、游标不前进；
- 429 / 5xx 重试、参数错误不重试、代理传递和未闭合 Bar 过滤；
- CSV 标准化、时间字段、重复、缺口和异常 OHLC；
- 冻结、Checksum、缓存复用、损坏检测和取消清理；
- Dataset ID 与 Symbol 防误配、引用保护和删除；
- NYSE 节假日、半日市、DST、强制离场边界和短窗口跳过；
- 独立窗口回测、保守强制平仓和报告持久化；
- 前端 TypeScript 检查与 Vite 生产构建。

每次发布前应执行：

```powershell
.\.venv\Scripts\python.exe -m pytest -q
cd frontend
npm run build
```

## 8. 当前边界

- 当前回测成交模型为保守 L0，不等价于真实盘口排队；
- 真实 Funding 序列尚未接入，前端明确禁用该选项；
- 进度流当前采用轮询，后续可在不改变任务模型的前提下增加 SSE；
- Walk-Forward 与成本敏感性属于补充稳定性检查；NYSE 主结果和窗口分布严格按独立休市窗口执行；
- 回测通过不代表策略可扩大真实资金，仍需样本外、模拟盘和小资金实盘验证。
