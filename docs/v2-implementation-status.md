# QuietGrid v2 改造交付与验证状态

> 最后更新：2026-07-17  
> 对照基线：`QuietGrid_v2_design/`  
> 本文是“代码已经做到什么、哪些能力仍受限、实盘前还缺什么”的事实清单，不是收益承诺。

## 1. 当前产品定位

QuietGrid v2 已从固定周末网格升级为：

> 以 Market Regime 为入场前置、以 Inventory Manager 管理未配对库存、以 Risk Manager 拥有最高否决权、以事件与快照支持审计和重放的自适应网格系统。

前端也不再是配置页或收益展示页，而是按任务组织的交易控制中心：

```text
运行：总览 → 市场状态 → 会话与库存 → 风险中心
研究：回测中心 → 策略重放
系统：记录与对账 → 策略设置 → 运维与验收
```

“易用”通过清晰的信息层级、渐进披露、统一危险操作确认和真实空状态实现，不通过删除专业能力实现。

| 页面 | 已实现的主要能力 |

| 风险中心 | 账户→窗口→会话/库存/相关性组预算，止损次数、杠杆边界、恢复条件、熔断操作 |
| 回测中心 | 数据集与参数表单、开发/验证/OOS 标签、冻结确认、报告列表与详情 |
| 策略重放 | 价格与网格线、Regime/Risk/Inventory 快照、事件游标、播放/暂停/逐事件/倍速 |
| 记录与对账 | 交易所挂单、本地订单、差异、成交和结构化日志 |
| 策略设置 | 常用草稿、差异、当前激活的完整 v2 政策；高风险政策只读并显示生效边界 |
| 运维与验收 | 连接/权限/费率/环境检查、有界运行、安全清扫和进程控制 |

响应式策略：

- 桌面端提供完整控制和研究能力；
- 移动端保留查看与紧急降风险操作；
- 移动端禁止高风险参数编辑；
- 状态同时使用文字、图标和颜色，避免只依赖颜色。

## 5. 回测中心实现

### 5.1 已实现

- 事件顺序与 `available_time` 校验，发现未来数据抛出 `LookAheadViolation`；
- `L0_CONSERVATIVE` 保守 K 线成交模型；
- 穿透 tick、同 Bar 风控优先、每 Bar 最大成交层数、确定性成交折扣；
- Maker/Taker 费用、止损滑点、资金费；
- 权益、回撤、库存利用率与成交明细；
- 样本标签：`DEVELOPMENT`、`VALIDATION`、`OOS_FROZEN`；
- OOS 参数冻结确认；
- Walk-Forward 折叠结果；
- Monte Carlo 的漏单、亏损放大和成本扰动；
- 固定窗口收益分布；
- 旧版报告读取时自动按权益曲线重建窗口盈亏，避免把测试本金误算为窗口亏损；
- 成交率、手续费和滑点敏感性；
- 数据区间、代码提交、参数版本、随机种子和原始 JSON 导出。

## 6. 明确未伪装为“已完成”的能力

以下能力需要更高质量数据或更完整基础设施，当前界面会明确显示不可用或禁用，不会生成虚假结论：

| 能力 | 当前边界 | 进入下一阶段的条件 |
|---|---|---|
| L1/L2 成交模型 | 当前仅有保守 L0 | 历史 bid/ask、主动成交量或队列数据 |
| 回测 Regime 混淆矩阵 | 普通 OHLC CSV 不含历史点差/深度 | 归档当时可得的 L1/L2 与特征快照 |
| 回测事件重放 | 当前重放真实/模拟运行事件 | 回测引擎输出逐事件订单意图、风控和撮合事件 |
| 反事实动作对比 | 按钮禁用并说明数据不足 | 保存“实际动作”和“严格系统动作”双轨事件 |
| 参数版本激活工作流 | 激活政策可查看，草稿可保存 | 增加验证报告绑定、审批和新窗口原子激活 |
| API 延迟/断线 Monte Carlo | 当前覆盖漏单、亏损和成本扰动 | 事件级执行模拟器和断线注入 |
| 运维备份恢复验收 | 尚未纳入控制台 | 备份、校验、恢复演练和验收记录 |
| Web 身份鉴权 | 当前仅保留部署警告与本地 CORS | 在任何外网或真实盘暴露前强制认证和权限分级 |

## 7. 发布与实盘门槛

当前代码和前端完成不等于可以扩大资金。满足以下全部条件前，不应进入真实资金放大：

1. 使用与目标交易环境一致的数据完成样本外和 Walk-Forward；
2. 若用于实盘，至少接入并验证 L1 成交模型，或对 L0 使用足够保守折扣；
3. OOS、最坏窗口和 Monte Carlo 尾部损失均在书面风险预算内；
4. 条件止损链路、进程重启恢复、断线重连、订单对账和安全清扫通过；
5. Web 启用鉴权，接口不直接暴露公网；
6. 仅使用可完全损失的极小资金、1 倍有效杠杆、单账户、单标的；
7. 多个独立窗口结束后均无残留挂单和仓位；
8. 实际成交、滑点和资金费与回测偏差可解释；
9. 所有扩大资金行为经过新的验证报告，不因短期盈利跳级。

## 8. 验证命令

```powershell
# 后端
.\.venv\Scripts\python.exe -m pytest

# 前端类型检查与生产构建
Set-Location frontend
npm run build
```

Windows 上长时间单进程运行全部异步测试可能触发系统 `WinError 10055`。这属于本机 socket buffer 耗尽；可按收集到的测试节点分批启动独立 pytest 进程，避免把环境资源问题误判为断言失败。

## 9. 使用原则

- 回测页用于验证假设，不用于证明策略必然盈利；
- 回测报告必须同时阅读成交模型、样本标签和成本假设；
- 风险中心比收益卡片优先；
- 运行时不得手工提高杠杆或库存上限；
- 任何订单差异、止损缺失或数据过期都应先暂停新增风险；
- 设计文档中的“计划”与本文件中的“已实现”必须保持区分。
