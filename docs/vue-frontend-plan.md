# QuietGrid Vue 前端控制台开发计划

> 阶段目标：在当前测试网链路已经可控运行的基础上，将只读 Streamlit 页面升级为 Vue 现代化控制台。前端不仅展示状态，还要能对正在进行的网格进行受控操作。

## 1. 产品定位

QuietGrid Vue 控制台是一个交易运维工作台，不是营销页。首屏直接进入可操作的运行视图，重点服务以下场景：

- 快速判断测试网/实盘运行是否健康。
- 查看活动网格、挂单、成交、波动率和系统日志。
- 对正在运行的网格执行明确、可回滚或可兜底的控制动作。
- 调整下一轮网格的关键参数，例如波动率算法、观察窗口、并发数和单标的开关。

## 2. 视觉与交互方向

设计系统采用高对比、数据密集、现代运维控制台风格：

- 主色：`#F59E0B`，用于主操作、焦点态和关键强调。
- 背景：`#0F172A`，深色控制台基底。
- 前景：`#F8FAFC`，保证文本可读性。
- 边框/分隔：`#334155`，用于低干扰区块边界。
- 危险操作：`#EF4444`，用于停止网格、强制平仓、清扫等动作。
- 字体：Inter 或系统 sans-serif，数据列使用等宽数字特性。
- 图标：统一使用 Lucide 风格图标，不使用 emoji 作为结构化图标。
- 交互：所有按钮有 loading、disabled、success/error 反馈；危险操作必须二次确认。

页面应避免大面积装饰渐变、营销式 hero 和卡片嵌套卡片。布局以工作台为主：顶部状态栏、左侧导航、主区域数据表/控制面板、右侧风险与日志摘要。

## 3. 技术架构

### 3.1 前端

- Vue 3 + TypeScript + Vite。
- Pinia 管理运行状态、配置草稿和控制动作状态。
- Vue Router 管理页面。
- ECharts 或 lightweight-charts 展示价格、波动率和 PnL 曲线。
- Lucide Vue 图标。
- CSS 使用 token 化变量，先用原生 CSS/Tailwind 二选一；第一阶段优先少依赖、可快速落地。

### 3.2 后端

保留 Python 交易进程。新增一个轻量 API 服务，向 Vue 提供：

- 只读状态 API：数据库摘要、活动会话、订单、成交、日志、测试网验证状态。
- 控制 API：启动/停止网格、暂停新开仓、执行安全清扫、更新策略配置草稿。
- 安全边界：所有危险动作后端二次校验，前端只提交意图，不直接操作交易所。

建议后端 API 使用 FastAPI，独立于交易 loop 进程启动，避免页面服务阻塞交易控制。

## 4. 页面信息架构

### 4.1 总览页

- 全局运行状态：测试网/实盘、loop 状态、最近心跳、活动会话数、开放订单数。
- 账户摘要：余额、可用余额、占用保证金、当前暴露。
- 风险摘要：剩余仓位、未清算法单、最近错误、费率健康。
- 快捷动作：启动有界测试、停止交易 loop、安全清扫、刷新验证。

### 4.2 网格控制页

- 活动会话表：标的、状态、网格上下沿、网格数量、步长、已实现盈亏、当前波动率。
- 单标的抽屉详情：挂单、成交、状态流转、波动率快照、风控事件。
- 控制动作：
  - 启用/停用某个标的。
  - 停止某个正在运行的网格。
  - 全局暂停新开仓。
  - 全局恢复新开仓。
  - 执行安全清扫。

### 4.3 策略参数页

- 交易参数：杠杆、单标的本金、最大并发、止盈阈值、止损缓冲。
- 观察参数：观察时长、K 线周期、最小样本。
- 网格参数：最小步长、最大网格数、资金费安全倍数。
- 波动率算法：
  - `std`
  - `parkinson`
  - `garman_klass`
  - `rogers_satchell`
  - `yang_zhang`
- 参数保存策略：
  - 第一阶段只保存为“下轮生效”的配置草稿。
  - 后续阶段再支持对部分运行中参数做受控热更新。

### 4.4 测试网验证页

- 测试网检查结果：连接、手续费、签名写接口、listenKey、持仓 smoke、安全清扫。
- 一键测试流程：选择运行时长，执行“前置持仓检查 -> 有界 loop -> 安全清扫 -> 后置持仓检查”。
- 展示每一步耗时、状态、错误信息和最终残留。

### 4.5 日志与审计页

- 系统日志、状态日志、控制动作审计日志。
- 支持按级别、模块、标的、时间过滤。
- 危险动作需要记录操作者、动作、参数、结果。

## 5. 控制 API 草案

第一阶段以最小闭环为目标：

```http
GET  /api/summary
GET  /api/sessions/active
GET  /api/sessions/{session_id}
GET  /api/orders?session_id=
GET  /api/trades?session_id=
GET  /api/logs/system
GET  /api/verification/testnet

POST /api/actions/testnet-run
POST /api/actions/safety-sweep
POST /api/actions/pause-new-entries
POST /api/actions/resume-new-entries
POST /api/actions/sessions/{session_id}/stop

GET  /api/config
PUT  /api/config/draft
POST /api/config/apply-next-run
```

危险动作统一请求体：

```json
{
  "confirm": true,
  "reason": "手动测试或风险处理",
  "request_id": "uuid"
}
```

## 6. 分阶段交付

### 阶段 A：Vue 项目骨架与静态控制台

- 建立 `frontend/` Vue 3 + TypeScript + Vite 项目。
- 完成布局、设计 tokens、导航、空状态、mock 数据。
- 页面包括总览、网格控制、策略参数、测试网验证、日志。
- 本阶段不接真实控制 API。
- 完成后提交并推送：`搭建 Vue 前端控制台骨架`。

### 阶段 B：只读 API 与真实数据接入

- 新增 FastAPI 只读 API。
- Vue 接入数据库摘要、活动会话、订单、成交、系统日志和测试网验证。
- 替代 Streamlit 只读看板的主要能力。
- 完成后提交并推送：`接入 Vue 控制台只读数据接口`。

### 阶段 C：测试网控制动作

- 接入安全清扫、一键测试网有界运行、暂停/恢复新开仓。
- 所有动作带二次确认、loading、结果 toast 和审计日志。
- 完成后提交并推送：`实现测试网控制动作面板`。

### 阶段 D：运行中网格控制

- 支持停止单个活动会话。
- 支持按标的启用/禁用下一轮开仓。
- 展示动作前后的订单/仓位确认结果。
- 完成后提交并推送：`支持活动网格控制操作`。

### 阶段 E：策略参数与波动率算法配置

- 支持编辑配置草稿。
- 支持选择波动率算法，并标注“下轮生效”。
- 展示当前运行参数和草稿差异。
- 完成后提交并推送：`实现策略参数和波动率配置`。

## 7. 验收标准

- `npm run build` 通过。
- Python 测试仍通过。
- 前端在 375px、768px、1440px 下无横向滚动和文字重叠。
- 所有危险动作都有二次确认和后端审计日志。
- 前端展示的活动会话数、开放订单数、测试网验证状态与数据库一致。
- 一键测试网流程结束后，页面能明确显示最终仓位和挂单残留为 0。

## 8. 提交与推送规则

从本阶段开始，每完成一个小阶段：

1. 跑对应测试和构建。
2. 使用中文提交信息。
3. 推送到当前远端分支。
4. 在交付说明中列出提交哈希、验证命令和结果。
